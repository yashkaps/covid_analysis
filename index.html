Relationship Between COVID-19 Cases and the Youth Population (by County)¶

Yash Kapoor¶

COVID-19 is a contagious desease caused by a coronavirus called SARS-CoV-2. It is believed to have originated in Wuhan, China, at the Huanan seafood market, but tracing the origin has proven to be quite difficult.

COVID-19 is thought to spread mainly through close contact from person to person, including between people who are physically near each other (within about 6 feet). People who are infected but do not show symptoms can also spread the virus to others. Older adults and people who have severe underlying medical conditions like heart or lung disease or diabetes seem to be at higher risk for developing more serious complications from COVID-19 illness.

Cases of reinfection with COVID-19 have been reported but are rare. As of December 2020, we are still learning about how the virus spreads and the severity of illness it causes.

A few of the most common symptoms of COVID-19 are

  • Fever or chills
  • Cough
  • Shortness of breath or difficulty breathing
  • Fatigue
  • Muscle or body aches
  • Headache
  • New loss of taste or smell
  • Sore throat
  • Congestion or runny nose
  • Nausea or vomiting
  • Diarrhea

More information about the virus can be found at the CDC website, as well as the WHO website. Many countries have their own dashboards set up to accurately track and document COVID-19 cases.

On March 11, 2020, the World Health Organization (WHO) announced the coronavirus outbreak a global pandemic.

About this analysis:¶

Since older adults are believed to be at higher risk of being infected by COVID-19, there were cases throughout the world where youths - many of them high school or college students - (inaccurately) believed that they were immune to COVID and completely disregarded safety guidelines (wearing masks, staying 6 feet away from other people, using hand sanitizer, etc.) and were involved in mass gatherings (partying, bars, crowded restaurants).

Older adults being at higher risk does not imply that young adults aren't at risk. They might be slightly more immune, but that makes them the perfect carrier since carriers do not display symptoms.

This analysis will try to investigate if there is a relationship between the number of COVID-19 cases and the youth population of a county. Throughout this analysis, "youth population" refers to young adults of ages 15 to 24. We will only be looking at the data for the United States in this analysis.

Data Collection¶

The first step of our analysis is gathering the data.

There are 2 kinds of data that we need to collect: data about the youth populations of each county of each state, and data about the number of cases over time for each county in each state.

The data about the populations will be extracted from the website of The Office of Juvenile Justice and Delinquency Prevention (OJJDP). As a federal office, the data from OJJDP is of course reliable, which is why we can use this data.

The data about number of cases over time, by county, will be extracted from the New York Times GitHub repository that houses COVID-19 data on not just county level, but also state and national levels. As one of the most popular news websites in the United States, the New York Times been known to achieve 70 million unique monthly visitors, outperforming other news channels like NBC News and The Washington Post. This gives us assurance about the accuracy and credibility of the data.

First we will extract the cases by county over time data.

We will only be reading the first 784,209 rows since they contain data up to November 30, 2020, which is enough for our purposes since we are examining past trends. If they choose to do so, anyone following this tutorial can remove the "nrows" argument to read all the available rows.

Some people might not be familiar with the FIPS column in the data, but it is just a "fancy" identifier for counties, and more accurate than ZIP codes in defining the boundaries of a county.

  • parsing them using a library to allow for easier and faster information retrieval
  • essentially converting an HTML table into a Python list
  • creating a dataframe out of said list
  • combining 51 such dataframes into a single dataframe that we can work with

# we'll create a dataframe for each state and store it in a list, then combine the 
# dataframes into a single dataframe at the end
df_list = [0]*51
j = 0
for i in sel_state_working_indices:
    params['selState'] = i     # changing the 'selState' property to specify the state
    r = requests.get(base_url, params=params)    # making the GET request
    soup = bs4.BeautifulSoup(r.text, 'html.parser')    # using BeautifulSoup to parse the HTML to make lookup easier
    txt = soup.find(id="tbldata").text  # finding the table, in HTML, that we want to scrape, and returning it as plain text
    # modifying the plain text so we can more easily extract relevant information from it
    txt = txt.replace("\n"," ")    
    txt = ([x.strip() for x in txt.split("\r") if x.strip() != ''])
    txt[0] = 'County'
    txt[1] = '2019'
    txt.insert(2,'Total')
    txt.insert(3,'All Counties')
    county_names = [x for i,x in enumerate(txt) if (i) % 3 == 0][1:]
    pop_2019 = [x for i,x in enumerate(txt) if (i+1) % 3 == 0][1:]
    # creating the dataframe for this particular state
    df = pd.DataFrame(list(zip(county_names,pop_2019)),columns=['County Name', 'Youth Population in 2019'])
    state_name = soup.find('h2').text.split(" ")[-1]   # extracting the state name from the webpage
    df['State'] = state_name  # creating a new column in the dataframe and setting every value equal to the state name
    df_list[j] = df   # adding the newely created dataframe to the list of dataframes
    j = j+1    # incrementing the dataframe list index

# combining all the dataframes into a single dataframe and resetting the index
youth_pop_all_states = pd.concat(df_list)
youth_pop_all_states.reset_index(drop=True)

Data Processing¶

After we have collected the data we require into DataFrames that we can work with, we need to process this data so it can better serve our purposes.

First we'll work with the cases by county over time dataframe. There are a few things we need to do here:

  • First we'll parse the 'date' column as dates, i.e. we'll convert them to a datetime format that Python and Pandas understands
  • We can then remove the 'day' part of the date and leave just the month, which will allow us to group by month then state and county so we can look at the number of cases/deaths per county per state as a function of time in months

  • We also need to realize that the data in this dataframe is cumulative (this is mentioned in the README on the GitHub repository), i.e. for a given date and location (by state and county), the number of COVID cases up to that point is stored in the cases column of the row. This means that we can represent each entire month by just looking at the last day.

  • We could also just directly do a group by on the state and county and look at a graph of total COVID cases vs. youth population (including both state and county is necessary to uniquely identify the county since multiple states can have counties with the same names).

Now that we are done with each of the DataFrames individually, it is time to combine them together into a single DataFrame that we'll call 'data_df'. We will be joining the cases and youth population datasets on the state and county columns. It is precisely for this step that we renamed the columns of the youth population dataframe.

This new DataFrame will have the columns 'county', 'state', 'youth_pop_2019', 'cases' and 'month'.

It may seem odd to include month in this analysis. After all, we're trying to see if a relationship exists between the youth population and the number of COVID cases in a county. We're including month as a parameter so we can utilize our data better and observe trends over time. After all, making any conclusions based on data for any single month would not be very credible.

First we'll reset the indices since any duplicate indices can cause trouble while merging the dataframes.

Then we'll do the merge on the 'state' and 'county' columns.

Note that this is an inner join which discards the rows where there is no match.

While this will result in a loss of some information, at the moment we have no way of filling in any gaps or making reasonable assumptions about the values.

If we had to make any assumptions, we could say that for the counties that are in the youth populations dataframe but not in the cases dataframe don't have a non-negligible number of COVID cases; in which case, dropping them won't result in any significant loss of information.

Note that since the data for both datasets originated from different sources, it is very possible that there were some county names that either had typos in one of the datasets or were recorded with slight differences for some reason. Our code isn't powerful enough to test for, detect and correct such subtleties so we have no choice but to leave those differences alone and just drop those rows.

Interpretation and Insight¶

Now that we've done some analysis on our data, it is time to draw meaningful conclusions from it.

In our case, it looks like there is indeed a relationship between the youth population (ages 15-24) and the number of COVID cases in a county.
